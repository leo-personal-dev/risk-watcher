
## What worked well
- The refinement of stories, acceptance criteria, and functional details to increase businness acceptance
- The refinement of the application, exploring details in best practices, code organization and architecture
- Increase the productivity to build the software application and tests
- Help to start things that are not well understooded or blocked about limited knowledge

## What didn't work
- If specifications are provided with high level of abstraction or too much ambiguity the output is not good and will demand more effort to validation
- Some logics to implement the data-driven approach are not fully implemented, demanding more iterations to optimize the solution

## What I'd do differently
- More details in specification and plan prompts
- More details in constitution file to create more design guard rails to improve the refinement
- Create a more elaborated goal model based on lean canvas to increase the specification generation (why and what) and plan specifications (how)
